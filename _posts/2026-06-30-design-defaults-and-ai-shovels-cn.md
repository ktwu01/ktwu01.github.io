我刚刚在火车上突然想起一件事：为什么互联网上的链接，几乎清一色都是那种饱和度极高的蓝色？

> 作者：[Koutian Wu](https://www.linkedin.com/in/ktwu01/)；[GitHub: ktwu01](https://github.com/ktwu01/)

你点开任何一个网页，超链接是蓝色的、带下划线的；点过之后变成紫色。保存按钮是一张软盘——一张今天的大学生可能根本没见过实物的软盘。邮件的图标是一个信封。删除是一个垃圾桶。我们对这些约定如此熟悉，以至于从来不觉得它们「需要一个理由」。

但它们当然有理由。而且关键在于：**这些理由在当年并不是显而易见的，它们是某一个具体的人、在某一个具体的时刻，做出的一个具体的默认选择（default）。** 只不过这个选择恰好流传了下来，成了我们今天误以为理所当然的「常识」。

这件事让我后背一凉。因为我们现在正在做的，构建 AI 的基础设施，本质上是同一件事——我们正在为一个还没有定型的世界设定 default。而这些 default，有的会像蓝色链接一样流传三十年，有的会像 `<blink>` 标签一样被人当作笑话扫进历史的垃圾桶。

那么问题来了：**哪些 default 活下来了，哪些死掉了，为什么？我们能不能从中读出，今天该怎么下注，才能成为那个「卖铲子的人」？**

---

## 一、那个蓝色，到底是怎么来的

先说蓝色链接本身。它并不是某个色彩心理学专家深思熟虑的结果。

1993 年的 Mosaic 浏览器（以及更早的一批超文本系统）需要一种方式，把「可以点击的文字」和「普通文字」区分开。当时大量显示器还是偏单色或低色深的，黑色已经被正文占用，而蓝色是在大多数屏幕上**既不是黑、又足够暗到能在白底上读清楚**的颜色。它不像红色那样有警告意味，不像绿色那样和「正确」绑定。蓝色几乎是当时唯一「安全」的选择。

换句话说，蓝色链接的胜出，靠的不是它「最好」，而是它「在那个约束条件下足够好，并且第一个被大规模铺开」。这是理解一切 default 的钥匙。

---

## 二、活下来的 default

我列一下那些设定于二三十年前、至今仍在统治我们屏幕的默认：

- **蓝色 + 下划线的链接**：已经成为「可点击」的通用语义。
- **软盘 = 保存**：能指（图标）和所指（实物）早已脱钩，但符号活了下来。
- **信封 = 邮件，垃圾桶 = 删除，购物车 = 结算**：拟物图标沉淀成了纯符号。
- **`@` 符号用于邮件地址**：1971 年 Ray Tomlinson 随手挑的一个键盘上「不会出现在人名里」的字符。
- **从上往下滚动、滚动条在右侧、`Ctrl+C / Ctrl+V`**：肌肉记忆级别的约定。
- **QWERTY 键盘布局**：它甚至不是为效率设计的（恰恰相反），却因为先发和锁定，统治了一个多世纪。

这些 default 的共同点是什么？

1. **它们被嵌进了协议、标准或肌肉记忆里**——迁移成本极高。换掉蓝色链接，你要重新教育全世界几十亿人。
2. **它们「足够好」，而不需要「最好」**——这就是 Lindy 效应：一个东西存在得越久，它继续存在下去的概率反而越大。
3. **它们的维持成本几乎为零**——没人有动力去动它，因为动它的收益远小于风险。
4. **网络效应**：因为别人都这么用，所以我也得这么用，于是更多人这么用。

---

## 三、被淘汰的 default

但同一个时代，也有大量「当年的理所当然」死得干干净净：

- **`<blink>` 和 `<marquee>` 标签**：闪烁和跑马灯，曾被认为很酷，今天是反面教材。
- **Flash**：曾经几乎等于「网页上的动态内容」，因安全、性能、封闭和移动端的不兼容而被整体抛弃。
- **Frames / 框架集**：曾是布局主流，被 CSS 取代。
- **拟物化设计（Skeuomorphism）**：早期 iOS 那种皮革、毛毡、木纹的质感，被扁平化设计取代。
- **访客计数器、留言簿、「点击进入」的 splash 页、"Best viewed in IE 6" 徽章**：整个一代审美的集体葬礼。
- **「所有链接都必须有下划线」**：注意，蓝色活了下来，但「强制下划线」这个搭档在很多现代设计里被弱化成了 hover 才显示。

为什么这些死了？我归纳出几条**淘汰的力量**：

1. **底层硬件/场景变了**：移动端的崛起一次性杀死了 Flash、框架集、和大量为鼠标设计的交互。
2. **更优解出现，且迁移成本可接受**：CSS 之于框架集。
3. **安全与性能的硬约束**：Flash 是被安全漏洞和电池续航判了死刑的。
4. **可访问性（Accessibility）和审美周期**：闪烁动画对某些人群是生理伤害；拟物化则纯粹是审美轮回。

对照第二节，结论很清晰：**一个 default 能不能活下来，几乎和它「当初设计得好不好」无关，而和它「是否被嵌入了不可迁移的底层、是否足够好到没人想动它、是否踩中了硬约束的雷」高度相关。**

---

## 四、把这套框架套到今天的 AI 上

现在，我们正在为 AI 设定一整批 default。它们此刻看起来天经地义，但其中一部分注定是 AI 时代的「蓝色链接」，另一部分注定是 AI 时代的 `<blink>`。我试着分一下类：

**可能成为「蓝色链接」的（嵌入了底层、迁移成本高、已有网络效应）：**

- **对话框 / Chat 作为人机交互的默认范式**：就像滚动条一样，正在变成肌肉记忆。
- **「OpenAI 兼容」的 API 格式**：它正在变成事实标准。无数推理服务、开源模型、网关都选择「兼容 OpenAI 的接口」，不是因为它最好，而是因为它**第一个被大规模铺开**——和蓝色链接的逻辑一模一样。
- **Token、上下文窗口、System Prompt 这套心智模型**：已经被写进了几乎所有开发者的大脑。
- **Markdown 作为模型输出的默认格式、工具调用（tool call）的 JSON Schema、以及 MCP 这类协议**：这些是「协议层」，一旦被嵌入生态，迁移成本会越来越高。

**可能成为 `<blink>` 的（追逐风口、停留在应用表层、随时被上层能力吃掉）：**

- 大量「套壳」应用：当底层模型每跨过一个能力阈值，就有一整批中间层产品像框架集被 CSS 取代一样消失。这一点我在[《我为何（部分地）改变了对"AI人才通胀"的看法》](/posts/2026/06/ai-talent-inflation-revisited-cn/)里已经亲身验证过——纯技术护城河会从脚下崩塌。
- 高度依赖某一代模型「怪癖」的 prompt 工程技巧。
- 为了演示而炫技、但没有踩中任何硬需求的交互形态——它们是这个时代的跑马灯。

## 五、所以，如何成为「卖铲子的人」

淘金热里真正稳定赚钱的，不是大多数淘金者，而是卖铲子、卖牛仔裤、卖水的人。这句话被说滥了。但结合上面这套 default 框架，我想说一个更精确的版本：

> **真正的「铲子」，是那个被所有人默认采用、却又最难被替换掉的 default 本身。**

谁定义了蓝色链接，谁就拥有了三十年。谁让自己的 API 格式变成「事实标准」，谁就拥有了整个生态的迁移惰性。NVIDIA 卖的是物理的铲子（GPU），但更深一层的铲子，是**协议、格式、标准、和被嵌入肌肉记忆的约定**。

那么具体怎么做？我给自己定了四条：

1. **去协议层和标准层下注，而不是应用表层。** 问自己：我做的这个东西，有没有可能变成别人「默认就这么用」的那个东西？还是说它只是这一代模型的临时补丁？
2. **追求「足够好 + 第一个铺开」，而不是「完美但太晚」。** 蓝色链接和 QWERTY 都证明了：先发 + 网络效应 >> 理论最优。在 AI 基建里，谁先让自己的接口被一百个项目集成，谁就赢了。
3. **主动制造迁移成本（用生态，而不是用锁死）。** 让别人接入你越久、周边工具越多，迁移走的痛感就越大——这是良性的，蓝色链接就是这么「锁定」全世界的。
4. **去解决那些「无聊但人人都要」的底层问题。** 评测、可观测性、推理网关、数据管线、Agent 之间的通信协议——这些不性感，但它们是软盘图标，是会沉淀下来的符号。

---

我从火车上的一个走神，想到的其实是一件让人有点敬畏的事：**我们今天随手设下的某一个 default，可能正在决定三十年后几十亿人的肌肉记忆。**

蓝色链接的设计者，当年大概并不知道自己在为人类定义一个跨越世纪的约定。而我们这一代做 AI 基建的人，则是清醒地站在了那个位置上。

既然如此，那就别去当第一千个淘金者了。去当那个，定义铲子长什么样的人。
